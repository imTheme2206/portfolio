# Approve the portfolio narrative and case-study evidence

Type: HITL

## What to build

Create and approve the content brief that will guide the overhaul: the intended audience, one clear positioning statement, the desired opportunities, the side projects that belong in the Project Playground, and the evidence that can be published for each entry. Record ownership, purpose, distinctive behavior, status, available destinations, and any publication boundaries so implementation does not invent claims.

## Acceptance criteria

- [ ] The primary audience and desired opportunity types are stated.
- [ ] A concise positioning statement communicates the value offered, not only the job title or technology stack.
- [ ] The complete side-project collection is selected without making one project the site's structural concept.
- [ ] Each project has an approved inventory covering purpose, ownership, notable behavior, media, status, stack, and destinations.
- [ ] Sensitive material has explicit publication and anonymization rules where applicable.
- [ ] The homepage narrative order is approved as Personality, Positioning, Project Playground, Experience, About, and Contact.

## Blocked by

None - can start immediately.
