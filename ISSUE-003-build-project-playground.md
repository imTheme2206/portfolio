# Build the reusable Project Playground

Type: AFK

## What to build

Create one consolidated showcase for every side project. On desktop, pair a sticky media theatre with naturally scrolling project chapters and direct project selection; on mobile, present the same content as visible media cards in normal flow. Concentrate the site's most expressive masking, progress, and preview motion in this section without making any one project the site's structural concept.

## Acceptance criteria

- [ ] Every side project appears in one collection with title, purpose, stack, preview, status, and available destinations.
- [ ] Desktop scrolling, pointer interaction, and keyboard focus can all select the active project.
- [ ] Project changes use one consistent masked media transition and visible progress indicator.
- [ ] No essential project information depends on hover or the custom cursor.
- [ ] Mobile uses natural document flow with a visible preview for every project.
- [ ] Adding a project through the shared data model automatically adds it to both desktop and mobile presentations.
